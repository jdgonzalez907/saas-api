# Go Use Case Generator

Generates use cases in Go following these rules:

## Structure Rules

| Element | Implementation |
|---------|----------------|
| Interface | Public, PascalCase noun/verb (e.g., `ChangeUserEmail`) |
| Struct | Private, camelCase (e.g., `changeUserEmail`) |
| Constructor | Returns interface, not struct |
| Execute | Single public method, receives context.Context, returns (output, error) |
| Dependencies | Private interfaces (repos, use cases), injected by constructor |
| Entities | NOT dependencies, loaded during execution |

## Required Elements

### File: `<name>.go`

```go
package <package>

import "context"

// Use Case Interface
type <UseCase> interface {
    Execute(ctx context.Context, input <Input>) (<Output>, error)
}

// Private Implementation
type <useCase> struct {
    repo      <Repository>      // Private dependency interface
    otherUC   <OtherUseCase>    // Private dependency interface
}

// Constructor - returns interface
func New<UseCase>(repo <Repository>, otherUC <OtherUseCase>) <UseCase> {
    return &<useCase>{
        repo:    repo,
        otherUC: otherUC,
    }
}

// Execute - only public method
func (uc *<useCase>) Execute(ctx context.Context, input <Input>) (<Output>, error) {
    // 1. Load entities from repos (not injected)
    // 2. Call entity methods (business logic)
    // 3. Call repository/use cases
    // 4. Wrap errors with use case error
    // 5. Return output
}

// Private auxiliary methods
func (uc *<useCase>) helperMethod() error {
    // Internal logic
}
```

### File: `<name>_test.go`

Mocks are generated by mockery and live in `./mocks/`. They can come from domain repos
(`./mocks/<domain>/`) or other use cases (`./mocks/<application>/`). Each test struct has
a `setup` field with signature `func(t *testing.T, repo *MockXxx, uc *MockYyy, ...)`. The
loop creates the mocks, calls setup, then invokes the use case.

```go
package <package>

import (
    "context"
    "errors"
    "testing"

    "github.com/jdgonzalez907/saas-api/internal/<domain>"
    mock_domain "github.com/jdgonzalez907/saas-api/mocks/<domain>"
    mock_application "github.com/jdgonzalez907/saas-api/mocks/<application>"
    "github.com/stretchr/testify/mock"
)

func Test<UseCase>_Execute(t *testing.T) {
    tests := []struct {
        name    string
        setup   func(t *testing.T, repo *mock_domain.Mock<Repository>, otherUC *mock_application.Mock<OtherUseCase>)
        input   <Input>
        want    <Output>
        wantErr error
    }{
        {
            name: "success",
            setup: func(t *testing.T, repo *mock_domain.Mock<Repository>, otherUC *mock_application.Mock<OtherUseCase>) {
                repo.On("FindByID", mock.Anything, int64(1)).Return(<entity>, nil)
                repo.On("Update", mock.Anything, mock.Anything).Return(nil)
                otherUC.On("Execute", mock.Anything, mock.Anything).Return(<result>, nil)
            },
            input:   <valid input>,
            want:    <expected output>,
            wantErr: nil,
        },
        {
            name: "error - <description>",
            setup: func(t *testing.T, repo *mock_domain.Mock<Repository>, otherUC *mock_application.Mock<OtherUseCase>) {
                repo.On("FindByID", mock.Anything, int64(1)).Return(nil, errors.New("database error"))
            },
            input:   <input>,
            wantErr: domain.Err<UseCase>,
        },
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            repo := mock_domain.NewMock<Repository>(t)
            otherUC := mock_application.NewMock<OtherUseCase>(t)
            tt.setup(t, repo, otherUC)
            uc := New<UseCase>(repo, otherUC)
            got, err := uc.Execute(context.Background(), tt.input)

            if !errors.Is(err, tt.wantErr) {
                t.Errorf("Execute() error = %v, wantErr %v", err, tt.wantErr)
                return
            }
            if tt.wantErr == nil && got == nil {
                t.Errorf("Execute() returned nil, want non-nil")
            }
        })
    }
}
```

**Key rules for tests:**
- Mocks come from `./mocks/` (generated by mockery), never hand-written
- Domain repos live in `./mocks/<domain>/`, use cases in `./mocks/<application>/`
- `setup` receives `(t, ...mocks)` and registers expectations via `.On().Return()`, does NOT return the mocks
- The test loop creates each mock with `mock_domain.NewMock<Repository>(t)` or `mock_application.NewMock<UseCase>(t)` (registers cleanup automatically)
- Use `mock.Anything` for arguments unless the test needs precise matching
- Use `mock.MatchedBy(func(x Type) bool { ... })` when the return value depends on the argument

## Rules

### Errors
- **Defined in entity/VO**: Errors belong to the entity/VO they pertain to
- **NotFound errors**: Defined in entity file (e.g., `ErrUserNotFound`)
- **Wrap dependency errors**: Use case wraps errors with its own error
- **Orchestration errors**: Defined in use case file for cross-entity logic
- **Ask developer**: If unclear which entity owns the error

**When to ask developer:**

| Situation | Question |
|-----------|----------|
| Error from single entity | "Should I define `ErrX` in the entity file?" |
| Error from multiple entities | "Should `ErrX` be in entity A, entity B, or use case?" |
| Orchestration error | "Should I define `ErrX` in the use case file?" |
| Cross-cutting error | "Where should I define this error?" |

```go
// Error defined in domain/user/user.go
var (
    ErrUserNotFound      = errors.New("user not found")
    ErrChangeUserEmail   = errors.New("cannot change user email")
)

// Orchestration error defined in use case file
var ErrTransferMoney = errors.New("cannot transfer money")

// Use case wraps errors using auxiliary method
func (uc *transferMoney) Execute(...) error {
    err := uc.repo.Update(ctx, user)
    if err != nil {
        return uc.wrapError(ErrTransferMoney, err)
    }
    return nil
}

// Private auxiliary method for error wrapping
func (uc *transferMoney) wrapError(useCaseErr, depErr error) error {
    return fmt.Errorf("%w: %v", useCaseErr, depErr)
}
```

### Validations
- **Entity/VO validations**: In entity or VO, not use case
- **Use case orchestrates**: Calls entity methods, handles results
- **Load entities during execution**: Not injected as dependencies

```go
func (uc *changeUserEmail) Execute(ctx context.Context, input ChangeUserEmailInput) error {
    // 1. Load entity from repo (NOT injected)
    user, err := uc.userRepository.FindByID(ctx, input.UserID)
    if err != nil {
        return uc.wrapError(domain.ErrChangeUserEmail, err)
    }

    // 2. Call entity method (validates internally)
    err = user.ChangeEmail(input.NewEmail, input.ExecutedBy)
    if err != nil {
        return err // Entity validation error
    }

    // 3. Persist
    err = uc.userRepository.Update(ctx, user)
    if err != nil {
        return uc.wrapError(domain.ErrChangeUserEmail, err)
    }

    return nil
}

// Private auxiliary method for error wrapping
func (uc *changeUserEmail) wrapError(useCaseErr, depErr error) error {
    return fmt.Errorf("%w: %v", useCaseErr, depErr)
}
```

### Dependencies
- **Private interfaces**: Not exported
- **Injected by constructor**: Never created inside use case
- **Never mutated**: Set once in constructor
- **Interface only**: Never struct dependencies
- **Allowed dependencies**: Repositories, other use cases
- **NOT dependencies**: Entities, VOs (loaded during execution)

```go
type UserRepository interface {
    FindByID(ctx context.Context, id int64) (*User, error)
    Update(ctx context.Context, user *User) error
}

type changeUserEmail struct {
    userRepository UserRepository // Interface, not struct
}
```

### Infrastructure
- **NEVER in use case**: No JSON, databases, HTTP, etc.
- **Only domain**: Call entities, VOs, or other use cases
- **Orchestration only**: Coordinate business logic

### Naming
- **Package**: Always in `usecase` package (or developer-specified package)
- **Business intention**: Name shows what business action
- **Verb + Noun**: `ChangeUserEmail`, `CreateOrder`, `DeactivateProduct`
- **Not technical**: Not `UpdateUserEmailUseCase`

## Scope

- **Create**: Generate new use case from scratch
- **Edit**: Modify existing use case (add logic, methods, dependencies)
- **Refactor**: Improve existing use case without changing behavior
- **Context**: ONLY `<name>.go` and `<name>_test.go`
- **External errors**: Developer must explicitly ask to fix

## Complete Example: TransferMoney Use Case

### Domain Entities

#### account.go
```go
package domain

import (
    "errors"
    "time"
)

var (
    ErrAccountNotFound        = errors.New("account not found")
    ErrInsufficientFunds      = errors.New("insufficient funds")
    ErrAccountInactive        = errors.New("account is inactive")
    ErrInvalidAmount          = errors.New("invalid amount")
    ErrCannotTransferToSelf   = errors.New("cannot transfer to self")
)

type Account struct {
    id      int64
    ownerID int64
    balance Money
    active  bool
}

type AccountDTO struct {
    ID      int64   `json:"id"`
    OwnerID int64   `json:"ownerId"`
    Balance MoneyDTO `json:"balance"`
    Active  bool    `json:"active"`
}

func NewAccount(id int64, ownerID int64, balance Money) (*Account, error) {
    if id == 0 {
        return nil, ErrAccountNotFound
    }
    return &Account{id: id, ownerID: ownerID, balance: balance, active: true}, nil
}

func (a *Account) ID() int64       { return a.id }
func (a *Account) OwnerID() int64  { return a.ownerID }
func (a *Account) Balance() Money  { return a.balance }
func (a *Account) Active() bool    { return a.active }

// Business methods
func (a *Account) CanWithdraw(amount Money) error {
    if !a.active {
        return ErrAccountInactive
    }
    if a.balance.LessThan(amount) {
        return ErrInsufficientFunds
    }
    return nil
}

func (a *Account) Withdraw(amount Money) error {
    if err := a.CanWithdraw(amount); err != nil {
        return err
    }
    a.balance = a.balance.Subtract(amount)
    return nil
}

func (a *Account) Deposit(amount Money) error {
    if !a.active {
        return ErrAccountInactive
    }
    a.balance = a.balance.Add(amount)
    return nil
}

func (a *Account) Equals(other *Account) bool {
    if other == nil {
        return false
    }
    return a.id == other.id
}

func (a *Account) ToDTO() AccountDTO {
    return AccountDTO{
        ID:      a.id,
        OwnerID: a.ownerID,
        Balance: a.balance.ToDTO(),
        Active:  a.active,
    }
}
```

### Use Case

#### transfer_money.go
```go
package usecase

import (
    "context"
    "errors"
    "fmt"

    "domain"
)

var ErrTransferMoney = errors.New("cannot transfer money")

type TransferMoney interface {
    Execute(ctx context.Context, input TransferMoneyInput) error
}

type transferMoney struct {
    accountRepo domain.AccountRepository
}

type TransferMoneyInput struct {
    FromAccountID int64
    ToAccountID   int64
    Amount        domain.Money
    ExecutedBy    int64
}

func NewTransferMoney(accountRepo domain.AccountRepository) TransferMoney {
    return &transferMoney{accountRepo: accountRepo}
}

func (uc *transferMoney) Execute(ctx context.Context, input TransferMoneyInput) error {
    // 1. Load both accounts
    fromAccount, err := uc.accountRepo.FindByID(ctx, input.FromAccountID)
    if err != nil {
        return uc.wrapError(domain.ErrAccountNotFound, err)
    }

    toAccount, err := uc.accountRepo.FindByID(ctx, input.ToAccountID)
    if err != nil {
        return uc.wrapError(domain.ErrAccountNotFound, err)
    }

    // 2. Orchestration validation (cross-entity)
    if fromAccount.Equals(toAccount) {
        return uc.wrapError(ErrTransferMoney, domain.ErrCannotTransferToSelf)
    }

    // 3. Call entity methods (business logic)
    if err := fromAccount.Withdraw(input.Amount); err != nil {
        return uc.wrapError(ErrTransferMoney, err)
    }

    if err := toAccount.Deposit(input.Amount); err != nil {
        return uc.wrapError(ErrTransferMoney, err)
    }

    // 4. Persist both accounts
    if err := uc.accountRepo.Update(ctx, fromAccount); err != nil {
        return uc.wrapError(ErrTransferMoney, err)
    }

    if err := uc.accountRepo.Update(ctx, toAccount); err != nil {
        return uc.wrapError(ErrTransferMoney, err)
    }

    return nil
}

// Private auxiliary method for error wrapping
func (uc *transferMoney) wrapError(useCaseErr, depErr error) error {
    return fmt.Errorf("%w: %v", useCaseErr, depErr)
}
```

### Key Points

| Error Type | Where Defined | Example |
|------------|---------------|---------|
| Entity validation | Entity file | `ErrInsufficientFunds` |
| Entity not found | Entity file | `ErrAccountNotFound` |
| Orchestration | Use case file | `ErrTransferMoney` |

**Error flow:**
1. Entity errors (`ErrInsufficientFunds`) wrapped with use case error
2. Orchestration errors (`ErrCannotTransferToSelf`) also wrapped
3. Repository errors wrapped with use case error
4. Caller receives wrapped error, can check with `errors.Is`

## Context Limits

- ONLY modify `<name>.go` and `<name>_test.go`
- Do NOT fix compilation errors in other files
- Do NOT modify callers or external code
- If other files have errors due to use case changes, developer must explicitly ask to fix them

## Workflow

1. **Ask developer**:
   - Which entities/VOs are involved?
   - Which repositories/use cases to call?
   - What is the business logic?
2. **Read existing files**: If `<name>.go` exists, read it and its test
3. **Read entity/VO files**: Validate required methods exist
4. **Read mocks**: Check `./mocks/<domain>/` and `./mocks/<application>/` for existing mock files; run `~/go/bin/mockery` if new interfaces need mocks
5. **Create or modify**: Generate/modify use case following rules
6. **Generate tests**: Create table-driven tests using mockery mocks, covering 100%
7. **Verify**: Ensure tests pass with `go test -v -run Test<UseCase>`

## Questions to Ask Developer

1. What entities/VOs does this use case use?
2. What repositories does it need?
3. What other use cases does it call?
4. What is the business logic step by step?
5. What errors should be returned?
